# 003 — Replace Daemon with roam

Status: done
Priority: medium

## Goal
Replace the hand-rolled JSON-RPC daemon with roam (https://roam.bearcove.eu/),
a service framework by fasterthanlime that generates typed clients for TypeScript
from Rust service definitions. This will eliminate the remaining serde_json usage
in eerie-daemon.

## Background
- roam lets you define services in Rust and generates TypeScript clients
- This eliminates the current serde_json envelope parsing in `eerie-daemon/src/rpc.rs`
- All data would go through facet-based serialization consistently
- roam is a newer project — check roam.bearcove.eu docs and crates.io for the
  latest API before implementing

## Tasks
- Check current roam version and API on crates.io / roam.bearcove.eu
- Define service interface in Rust (file ops, circuit parsing, simulation)
- Generate TypeScript client using roam
- Update `src/main/index.ts` to use the generated roam client instead of manual JSON-RPC
- Remove `serde_json` from `eerie-daemon/Cargo.toml` (last serde usage in the project)
- Update workspace Cargo.toml to remove serde_json if no longer needed anywhere

## Files to modify
- `eerie-daemon/src/main.rs`
- `eerie-daemon/src/rpc.rs` (likely delete and replace)
- `eerie-daemon/Cargo.toml`
- `src/main/index.ts`
- Root `Cargo.toml` (workspace deps)

## Acceptance criteria
- `eerie-daemon` has zero serde/serde_json dependencies
- TypeScript client is generated by roam (not hand-written)
- File open/save, circuit parse, and DC simulation all work through the new roam interface
